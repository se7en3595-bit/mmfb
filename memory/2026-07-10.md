## Excel 预览文字不可见 + 对比度调节 (2026-07-10 晚)

### 问题诊断
用户反馈："调整对比度，excel拖入预览看不到文字"

根本原因定位：
1. openpyxl 读取 Excel 单元格时，字体颜色为"自动"(Automatic) 返回 `"00000000"` (ARGB 全透明黑)
2. 原 Python `_extract_style` 未过滤此值，直接传给前端
3. 前端 `_formatColor` 截取 8 位 hex 后 2 位 alpha → 剩余 `000000` (纯黑)
4. 结果：`td.style.color = "#000000"` 覆盖了 CSS 变量 `--color-text-primary`
5. 在 Dark 主题（深色背景）下，显式设置的纯黑文字与深色背景融为一体 → 看不见

同样问题也影响背景色：原代码只过滤了 `bgColor` 的 `"00000000"`，未过滤 `color` 字段。

### 修复方案

**三处修改：**

1. **Python 后端** (`mmfb/handlers/xlsx_handler.py` L195-233):
   - 新增 `_is_auto_color(raw)` 辅助函数，识别 `"00000000"` / `"000000"` / `""` 三种自动标记
   - `style["color"]` 和 `style["bgColor"]` 写入前均调用此函数过滤
   - 无效颜色 → 不写入 style 对象 → 前端无 inline style → 回退到 CSS 主题变量

2. **前端 JS** (`mmfb/frontend/js/xlsx_viewer.js`):
   - `_formatColor` 方法增加 ARGB alpha 通道判断：`alpha === '00'` → 返回空字符串
   - 新建 `_applyFilter` 方法：对 `bodyEl` 应用 CSS `filter: brightness() contrast()`
   - 新建 `_showAdjustDialog` 方法：浮动弹窗，含两个 range 滑块（亮度/对比度，范围 0.3~2.0）
   - 工具栏追加 `◐` 按钮，点击弹出调节面板
   - 滑块 input 事件实时预览（不请求后端），点击"重置"一键恢复 1.0

3. **CSS** (`mmfb/frontend/css/xlsx_viewer.css`):
   - 新增 `.xlsx-viewer__adjust-btn` 按钮样式
   - 新增 `.xlsx-adjust-dialog` 及其子元素样式（浮动面板布局、滑块、按钮）

### 技术选择理由
- CSS filter 方案：GPU 加速合成层，对 10000 单元格 table 无重排开销
- 不做像素级编辑：Excel 预览器以"查看+单元编辑"为主，亮度/对比度属于"显示补偿"，持久化无意义
- Python + 前端双重过滤：后端保证数据干净（其他前端消费者也受益），前端作为防御性渲染兜底

### 涉及修改的文件
1. `mmfb/handlers/xlsx_viewer.py` - _extract_style 方法
2. `mmfb/frontend/js/xlsx_viewer.js` - _formatColor + _applyFilter + _showAdjustDialog + _buildStructure 工具栏
3. `mmfb/frontend/css/xlsx_viewer.css` - 按钮样式 + 弹窗样式

## Excel 文字仍不可见（第二次修复）(2026-07-10 晚)

### 问题诊断
用户反馈："字体可以切换黑白色 还是看不清楚"

根因：第一次修复只处理了 `00000000`（ARGB 自动色），但 Excel 中大量单元格使用**显式纯黑** `FF000000`（Excel 默认文字色就是 ARGB 纯黑）。这个值通过过滤后变成 `#000000`，在 Dark 主题深色背景上依然看不见。

### 修复方案
**一处修改：**

1. **前端 JS** (`mmfb/frontend/js/xlsx_viewer.js`)：
   - `_formatColor` 方法增加纯黑文字判断：6 位 hex 为 `000000` 且用于文字（非背景）时返回空字符串
   - 让 CSS 主题变量 `--color-text-primary` 接管纯黑文字色
   - Light/Warm 主题下 CSS 变量本身就是近黑色，视觉无差异
   - Dark 主题下 CSS 变量是浅色文字（#E8E8EC），深色背景上立刻清晰

### 技术选择理由
- 纯黑在 Excel 中语义就是"默认文字色"，应跟随显示环境
- 纯黑背景色不受影响（isBg 参数区分）
- 不修改 Python 后端，纯前端渲染层修复

### 涉及修改的文件
1. `mmfb/frontend/js/xlsx_viewer.js` - _formatColor 方法追加纯黑文字判断

## 文字颜色切换面板 (2026-07-10)

### 问题
用户反馈"绿色背景下看不到白色字体"：在 Excel 中手动设置绿色背景 + 白色字体后，预览器仍按原始颜色渲染，对比度不够看不清。

### 修复方案
新增"文字颜色切换面板"。

**4 处修改：**

1. **构造函数** (`mmfb/frontend/js/xlsx_viewer.js`)：新增 `this._textColorOverride = null` 状态
2. **工具栏按钮** (`_buildStructure`)：新增 `A` 字符按钮（.serif + 粗体），点击调用 `_showColorPanel()`
3. **新方法**：
   - `_applyTextColor()`：有 override 时遍历 `_domCellMap` 逐个覆盖 `style.color`；为 null 时重新渲染恢复原始
   - `_showColorPanel()`：8 色调色板（默认/黑/白/红/蓝/绿/橙/紫），点选后存状态并调用 `_applyTextColor()`
4. **CSS** (`mmfb/frontend/css/xlsx_viewer.css`)：
   - 工具栏加 `position: relative`
   - 新增 `.xlsx-viewer__color-btn` / `.is-override` 样式（底部色条反映当前选中色）
   - 新增 `.xlsx-color-panel` 选择面板样式（3×3 grid + 右上浮动）

### 技术选择
- override 状态只影响渲染层（DOM 颜色），**不影响底层数据和保存**
- 切换"默认"走重新渲染路径（`_renderSheet`），逻辑最简单，避免维护两份样式
- `_applyTextColor()` 在每次 `_renderSheet` 调用后自动重切到 override 态，无需手动挂接到每个调用点
- 面板关闭走 `mousedown` 外部点击检测（与亮度/对比度对话框一致）

## 文字颜色面板弹不出来 (2026-07-10 第三次修复)

### 问题
用户反馈："无调色板弹出"

根因：**两个 bug 叠加**：

1. **挂载位置错误**：`_showColorPanel` / `_showAdjustDialog` 用 `root.appendChild(panel)` 把面板挂到 `.xlsx-viewer` 根节点。根节点没有 `position:relative`，面板 `position:absolute; top:calc(100%+4px); right:0` 找不到定位祖先，最终渲染在屏幕外（跟着最近的有定位的祖先走了）。

2. **关闭逻辑干扰**：关闭面板的 `mousedown` 外部点击监听器没有排除触发按钮本身。点击 A 按钮时：`mousedown` 事件在 `click` 之前触发，已被外部监听器捕获 → 先 `panel.remove()` 关闭，此时 `click` 再触发 `_showColorPanel()` 发现没有旧面板 → 又新建一个，表现为"关了又开"的闪烁或卡死。

### 修复方案
**5 处修改：**

1. `_showColorPanel`：`var host = this._root.querySelector('.xlsx-viewer__toolbar-right')` + `host.appendChild(panel)` + `host.querySelector` 查旧面板
2. `_showColorPanel` 外部 mousedown 监听：加 `if (self._colorBtn && self._colorBtn.contains(ev.target)) return;` 排除按钮
3. `_showAdjustDialog`：同上，`host` 引用 + `host.appendChild(dlg)`
4. `_showAdjustDialog` 外部 mousedown 监听：加 `if (self._adjustBtn && self._adjustBtn.contains(ev.target)) return;` 排除按钮
5. CSS：`.xlsx-viewer__toolbar-right` 加 `position: relative`，作为两个面板的定位祖先

### 涉及修改的文件
1. `mmfb/frontend/js/xlsx_viewer.js` - 2 个方法的挂载目标 + 外部点击排除逻辑
2. `mmfb/frontend/css/xlsx_viewer.css` - `.xlsx-viewer__toolbar-right` 加 `position: relative`
