### 今日全部 bug 修复总结

本次会话共修复 5 个用户报告的问题，**已全部完成**：

#### Bug 1：拖入 PDF 一直显示"正在加载 PDF"
**状态：✅ 已完成（之前修复）**

**根因**：`pdf_viewer.js` 原本使用 `file://` URL 直接加载，触发 Chromium CORS 策略。

**修复方案**（turn 39-48）：
- 改用 `MMFBBridge.api.readFileBase64(path)` 通过 Python bridge 读取文件为 base64
- 前端解码为 ArrayBuffer 后传给 PDF.js
- 相关文件：`mmfb/frontend/js/pdf_viewer.js` 第 155-227 行

**额外说明**：`.PDF.TXT` 现象是 Windows 系统设置导致的用户行为，不是程序 bug。

---

#### Bug 2：分屏模式拖入文件无反应
**状态：✅ 已完成（本轮最终修复）**

**根因**：`SplitView` 的 webview 设置了 `setAcceptDrops(False)`，拖拽被禁用。

**修复方案**：
- `SplitView` 启用自身拖拽接收：`self.setAcceptDrops(True)` (split_view.py 第 72 行)
- 实现 `dragEnterEvent` 和 `dropEvent` (第 74-104 行)
- 拖入文件后自动加载到左栏，第二个文件作为对比加载到右栏

---

#### Bug 3：格式转换报错 "self.root.getElementById is not a function"
**状态：✅ 已完成（本轮最终修复）**

**根因**：`conversion_viewer.js` 中多处误用 `this.root.getElementById` / `self.root.getElementById`，但 `this.root` 是 DOM 元素，没有 `getElementById` 方法（这是 `document` 的方法）。

**修复方案**：
- 全文件中所有 `*root.getElementById` 调用已修正为 `document.getElementById`
- 文件位置：`mmfb/frontend/js/conversion_viewer.js`（多次出现）

---

#### Bug 4：关闭按钮行为不正确
**状态：✅ 已完成（本轮最终修复）**

**根因**：
- 关闭按钮 (`closeEvent`) 原本被 `ignore()` 并隐藏到托盘，用户期望真正退出
- 最小化按钮 (`showMinimized`) 也隐藏到托盘，用户期望最小化到任务栏

**修复方案**：
1. **关闭按钮**：`closeEvent` 移除隐藏逻辑，直接调用 `super().closeEvent(event)` 退出 (window.py 788-795 行)
2. **最小化按钮**：`showMinimized` 简化为 `super().showMinimized()`，恢复正常最小化到任务栏的行为 (window.py 766-769 行)

---

#### Bug 5：main.py 打开闪退
**状态：✅ 已完成（本次新发现并修复）**

**根因**：
1. `window.py` 第 766 行 `showMinimized` 函数缩进错误（使用 2 个 tab 而非 1 个）
2. `closeEvent` 函数混用了空格和 tab 缩进，导致 Python 解析失败
3. **最关键的崩溃原因**：`window.py` 在**模块顶层**导入了 `QWebEngineView`（第 49 行），这发生在 `QApplication` 创建之前，违反了 QtWebEngine 的初始化要求，直接在 C++ 层崩溃

**修复方案**：
1. 修正 `showMinimized` def 行缩进为单 tab (window.py 766 行)
2. 将 `closeEvent` 的 4 空格缩进统一改为 tab 风格 (window.py 784-791 行)
3. **延迟导入 QWebEngine 相关类**：
   - 移除顶层导入 `QWebEngineView` 和 `QWebEnginePage, QWebEngineSettings` (第 49-50 行)
   - 在 `_setup_webengine()` 方法内部延迟导入，确保在 `QApplication` 创建后才初始化 QtWebEngine
   - 同时将 `QWebEngineView` 的创建也移到 `_setup_webengine()` 中

**涉及文件**：
- `mmfb/core/window.py` - 缩进修复 + 延迟导入 QtWebEngine
- 历史修复文件：`mmfb/frontend/js/pdf_viewer.js`, `mmfb/frontend/js/conversion_viewer.js`, `mmfb/core/split_view.py`
