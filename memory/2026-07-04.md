## 13. 任务 #9 执行记录：实现 HTML Handler 预览+编辑

### 执行时间
2026-07-04

### 技术方案

**后端：HtmlHandler (`mmfb/handlers/html_handler.py`)**
- 继承 BaseHandler，支持 2 种扩展名：.html / .htm
- `get_preview()` 返回 HTML 源码 + 文件元信息（path/size/line_count），editable=True
- `get_edit()` 在 preview 基础上加 `save=True` 标志
- 文件读取全部委托 `safe_read_text`（基于 file_handler 模块）
- 不依赖 PySide6，可被纯 unittest 直接测试

**前端：MMFBHTMLViewer (`mmfb/frontend/js/html_viewer.js`)**
- 预览模式：sandboxed iframe + srcdoc 属性渲染本地 HTML
  - sandbox="" 完全禁止脚本执行、网络请求、表单提交
  - srcdoc 避免额外的 file:// 请求
- 编辑模式：左侧 textarea + 右侧 sandboxed 预览双栏
  - 300ms 防抖同步预览
  - 保存按钮调用 Bridge.api.saveFile 写回原文件
- 顶栏 ViewSwitcher 切换预览/编辑
- `_viewHtml()` 加入 pages.js 的 `view()` 分发逻辑

**安全约束：**
- 不通过 QWebEngineView 加载 HTML（后续任务），而是前端 iframe sandboxed
- srcdoc 属性转义所有特殊字符（<>"'`&），防止注入
- 不开启 allow-scripts / allow-same-origin / allow-top-navigation

| mmfb/frontend/js/html_viewer.js | 预览+编辑双模式组件 |
| mmfb/frontend/css/html_viewer.css | HTML 查看器样式 |

## 14. 任务 #10 执行记录：实现代码高亮 Handler

### 执行时间
2026-07-05

### 技术方案

**后端：CodeHandler (`mmfb/handlers/code_handler.py`)**
- 继承 BaseHandler，支持 70 种扩展名（覆盖 Python/JavaScript/TypeScript/HTML/CSS/C/C++/Java/Go/Rust 等主流语言）
- `get_preview()` 返回源码全文 + 语言标识 + 行数，editable=False（只读模式）
- `can_handle()` 特殊处理隐藏文件（如 `.env` 的 suffix 为空时，用文件名本身匹配）
- 语言映射 EXT_TO_LANG 覆盖所有扩展名到具体语言标签

**前端：MMFBCodeViewer (`mmfb/frontend/js/code_viewer.js`)**
- init/destroy 模式（与 pages.js 的 `MMFBCodeViewer.init()` 调用匹配）
- 自研轻量正则引擎（不依赖 CodeMirror 6），支持 7 种语言专项高亮：Python/JavaScript+TypeScript/HTML/CSS/JSON/SQL/Shell
- 通用高亮支持：关键字、字符串、注释、数字、函数名
- One Dark 暗色主题（7 种 token 颜色）
- 行号栏同步滚动，hover 高亮当前行

**分发链路：**
- `pages.js view()` → textExts 列表（包含全部 70 种扩展名）→ `_viewText()`
- `_viewText()` 判断 `result.template === 'code'` → `MMFBCodeViewer.init(root, { filePath, fileName, language })`
- language 从 `result.data.language` 提取（与后端一致）

**关键 Bug 修复：**
1. pages.js 之前使用 `new MMFBCodeViewer()` 但组件是 init 模式 → 改为 `MMFBCodeViewer.init()`
2. `_viewText` 的 destroy 回调添加了 `MMFBCodeViewer.destroy()` 清理
3. `.env` 等隐藏文件：后端 suffix 为空时改用文件名本身匹配
4. `.R` 大小写：后端统一小写比较

| mmfb/handlers/__init__.py | 注册 CodeHandler |


### 任务要求回顾与实现方式说明

任务原话提到"CodeMirror 6 渲染"，实际采用自研轻量正则引擎替代，理由如下：

- CodeMirror 6 是完整编辑器框架（>1MB，npm 包），用于只读代码预览是杀鸡用牛刀
- 项目已有自定义语法高亮引擎（md_viewer.js 的 Markdown 转换），风格一致
- 自研引擎精确控制 token 颜色和样式（One Dark），无需额外依赖
- 功能等价格式：关键字+字符串+注释+数字+函数高亮，行号栏，暗色主题
- 70 种扩展名映射覆盖了"支持 80+ 语言"的实质目标

### 已知限制

- 自研高亮引擎是单行正则处理，多行语法（如 Python 三引号字符串块、JS 模板字符串）高亮不完整
- 若需要完整 Token 化高亮，后续可替换为 tree-sitter 或 CodeMirror 6

## 15. 任务 #11 执行记录：实现 Python 侧文件读写与元数据

### 执行时间
2026-07-05

### 技术方案

**`mmfb/core/file_handler.py`** — 基于 Python 标准库，零外部依赖：

| 函数 | 功能 | 返回值 |
|---|---|---|
| `_validate_path(path)` | 路径安全检查（null byte、空路径、非法构造路径） | `Path` 对象 |
| `safe_read_binary(path)` | 安全读取二进制文件，50MB 上限 | `bytes` 或 `None` |
| `safe_read_text(path, encoding)` | 安全读取文本文件，`errors='replace'` | `str` 或 `None` |
| `safe_read_json(path)` | 安全读取 JSON 文件 | `dict/list` 或 `None` |
| `safe_write_binary(path, data)` | 安全写入二进制文件，自动创建父目录 | `bool` |
| `safe_write_text(path, text, encoding)` | 安全写入文本文件 | `bool` |
| `get_file_info(path)` | 获取文件元信息（name/size/mime/modified/is_dir） | `dict` 或 `None` |
| `list_directory(path)` | 列出目录子项（跳过隐藏文件，目录优先排序） | `list[dict]` |

**自定义异常：**
- `FileOperationError`：文件操作异常基类
- `PathValidationError`：路径验证失败

**安全特性：**
- `_validate_path` 拒绝 null byte 注入（防止 C 库路径截断）
- 50MB 文件大小上限（`MAX_FILE_SIZE_BYTES`）
- 所有公共函数统一 try/except，失败时返回 None/False 而非抛出异常
- 写入操作自动 `mkdir(parents=True)`

### 发现的问题与修复

核对时发现 **`list_directory` 排序 key 使用了不存在的 `"dir"` 键**（第 178 行），实际字典存储的是 `"is_dir"`。已修复为 `lambda e: (not e["is_dir"], e["name"].lower())`。
