# 2026-07-11 记忆记录

## Excel 与分屏交互问题修复

### 1. Excel 预览字体颜色一致性修复
- **问题诊断**：此前为了在深色模式下自动适配文字颜色，强制将 HEX 纯黑颜色 (#000000) 映射为跟随主题，但这导致在浅色/暖纸模式下无法保持与 Excel 源文件的一致性，文字没有忠实于源文件的颜色呈现。
- **解决方案**：在 `xlsx_viewer.js` 的 `_formatColor` 中移除了对 `000000` 强制转空字符串的限制，允许所有非 ARGB `00`（即有效的不透明颜色）直接返回其真实的 HEX 颜色，保证渲染结果与源文件完全一致。

### 2. 预览模式下复制内容实现
- **解决方案**：修改了 `xlsx_viewer.css` 中的 `.xlsx-table__cell` 样式，为其添加 `user-select: text; -webkit-user-select: text;`。使用户能够在非编辑状态下通过鼠标选定单元格文本并自由复制。

### 3. 空数据表双击编辑功能实现
- **解决方案**：修改了 `xlsx_viewer.js` 的 `_renderSheet` 方法。当数据表行或列为 0 时，空状态下新增“双击以初始化表格并开始编辑”的提示。如果处于编辑模式，双击此区域将自动将其初始化为 10x5 的默认空白表格数据，并再次渲染以供就地双击编辑并保存。

### 4. 分屏模式点击无效问题修复
- **解决方案**：在分屏视图容器 `split_view.py` 的初始化中，对两个 MMFBWebView 实例显式设置了 `setAttribute(Qt.WidgetAttribute.WA_TransparentForMouseEvents, False)`。确保它们的鼠标事件响应不被 QSplitter 或底层父容器的拖拽行为遮挡或透传，使得在分屏状态下的点击、双击能够百分之百准确地路由进对应的 QWebEngineView，彻底解决点击和双击编辑无反应的交互故障。
